UI Primitives — renderable components, injectable into any view
indicator icon or value beside a BNA unit → on Outline   ·   highlight colored range over text → on Writing   ·   card structured content block → on Panel

Figure N. Function registration end-to-end. ① The foundation exposes two interfaces: a DocumentSnapshot containing the complete coordination space state (BNA units, dependencies, writing sections, interaction history, prior function results), and a UI surface with tagged primitive slots across three views (outline: indicator, expandable; writing: highlight, side-note; panel: card, input). ② Two functions registered independently: check-drift (AI prompt) reads BNA units and writing sections, outputs coverage and sentence alignments, and declares rendering to indicator and highlight; check-terms (local JS) reads writing sections, outputs term mismatches, and declares rendering to highlight and card. ③ After registration, each function's output fills its declared UI slots — the same tag system connects function declarations to rendered views.
