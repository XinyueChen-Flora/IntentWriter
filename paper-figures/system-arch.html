Figure N. The GroundingKit data model. The Protocol Layer contains three registration types: SenseProtocol (bundles functions with triggers), Gate (reads function results and routes by conditions), and NegotiateProtocol (defines roles, actions with optional step sequences, deliberation stages, and resolution rules). Colored items show built-in registrations; community developers register through the same API. The Function Layer provides atomic capabilities that declare an executor, output schema, and display bindings — functions are shared across protocols and can chain by reading stored results. The Foundation is platform-defined: DocumentSnapshot assembles the full coordination space state as input to every function; FunctionResult stores outputs for chaining and Gate evaluation; ViewPrimitive provides the rendering surface; Proposal tracks negotiation state.
